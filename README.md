# 시설 점검/유지보수 기록 관리 페이지

내부 직원용 시설 점검 및 유지보수 이력을 관리하는 웹페이지입니다.

## 포함된 기능
- 시설명/위치, 점검일자, 점검자, 상태, 점검 내용, 다음 점검 예정일 기록
- 상태별(정상/이상발견/수리중/수리완료) 필터링
- 시설명/점검자 검색
- CSV로 내보내기 (엑셀에서 바로 열림)

## 지금 상태 (중요)
현재 버전은 데이터를 **브라우저의 localStorage**에 저장합니다.
즉, **나만 볼 수 있고 다른 직원과 공유되지 않습니다.** 우선 이 상태로 GitHub Pages에
올려서 화면과 기능을 확인해보시고, 여러 명이 데이터를 공유해야 한다면 아래
"3. 여러 직원과 데이터 공유하기" 단계를 진행해주세요.

---

## 1. GitHub 저장소(Repository) 만들기

1. github.com 로그인 → 우측 상단 `+` → `New repository`
2. Repository name 입력 (예: `facility-manager`)
3. Public 선택 (Pages 무료 사용을 위해 Public 권장)
4. `Create repository` 클릭

## 2. 파일 올리기 및 GitHub Pages 켜기

### GitHub Desktop을 쓰는 경우
1. 방금 만든 저장소를 `Clone` (GitHub Desktop에서 `File > Clone repository`)
2. 로컬 폴더에 이 프로젝트의 `index.html`, `README.md` 파일을 복사해서 넣기
3. GitHub Desktop에서 변경사항 확인 → 커밋 메시지 입력 (예: "첫 페이지 업로드") → `Commit to main`
4. 우측 상단 `Push origin` 클릭

### 웹에서 직접 올리는 경우 (더 간단함)
1. 저장소 페이지에서 `Add file > Upload files` 클릭
2. `index.html` 파일을 끌어다 놓기
3. 하단 `Commit changes` 클릭

### GitHub Pages 활성화
1. 저장소 상단 메뉴 `Settings` 클릭
2. 왼쪽 메뉴에서 `Pages` 클릭
3. `Branch`를 `main`, 폴더는 `/ (root)` 선택 후 `Save`
4. 1~2분 후 `https://내계정.github.io/facility-manager/` 주소로 접속 가능

## 3. 여러 직원과 데이터 공유하기 (선택, 추천)

localStorage는 브라우저(=개인 PC)별로만 저장되기 때문에, 여러 직원이 같이 입력하고
보려면 공용 저장소가 하나 필요합니다. 코딩 지식이 많지 않아도 다룰 수 있는
**Google Sheets + Google Apps Script** 조합을 추천드립니다. (완전 무료)

### 대략적인 흐름
1. Google Sheets에 새 시트를 만들고, 헤더(시설명, 점검일자, 점검자, 상태, 내용, 다음점검일)를 입력
2. 해당 시트에서 `확장 프로그램 > Apps Script` 열기
3. 시트에 데이터를 읽고 쓰는 간단한 스크립트를 작성 후 "웹앱으로 배포"
4. 배포된 웹앱 주소(URL)를 `index.html`의 저장/조회 부분에 연결

이 부분은 실제 구글 계정과 시트가 있어야 정확히 진행할 수 있어서, 원하시면
**다음 단계로 Google Apps Script 코드를 같이 작성**해드릴 수 있습니다. 구글 시트를
만들 준비가 되면 말씀해주세요.

### 대안: 더 간단하지만 제약 있는 방법
- 직원 수가 적고(2~3명), 매번 파일로 주고받아도 괜찮다면: 위의 "CSV 내보내기"
  기능으로 각자 내보낸 뒤 한 파일로 합치는 방식도 가능합니다 (수작업 필요).
